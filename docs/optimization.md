# 性能与优化方案

分四块:在线生成速度、高并发、初始化提速、同步管线。标 ✅ 的已在代码中实现,标 ⬜ 的为本轮待做,标 🔭 的为后续可选。

## 1. 在线生成速度(目标 p95 <100ms)

| 项 | 状态 | 说明 |
|---|---|---|
| 池内随机选址走索引 | ✅ | `random_key` 轴心 + `(country, *_key, active, random_key)` 复合索引,LIMIT 64 截断 |
| 先取 id 再批量取 runtime 行 | ✅ | 避免视图全列扫描,IN 批查替代逐行回表 |
| 国家计数 30s 内存缓存 | ✅ | `/countries`、`/data-health` 不再每次全表 GROUP BY |
| 前端预取队列 | ✅ | 每个筛选组合预取 5 条,点击"随机生成"零网络等待 |
| 读时富化替代运行时翻译 | ⬜ | admin1 码→三语全名、locality 中文名从 catalog 直查(每次 1-2 个 LIMIT 1 查询),不调用翻译 API |
| 回退链坐标查询用 RTree | ⬜ | 城市无精确匹配时按目录坐标做半径查询,25km→100km 两档,索引命中 |
| 地图/静态资源缓存头 | ✅ | 目录接口 max-age=30d,国家列表 7d |

## 2. 高并发与异步

**读路径**:SQLite WAL 模式天然支持多读者并发、读写不互斥;API 为单进程事件循环,纯内存+索引读,单核可支撑数百 QPS,远超使用规模。若未来需要横向扩展:同一 sqlite 文件可开多个只读连接/多个 Node worker(🔭)。

**写路径**:全库单写者 = 同步进程。每国导入在单事务内完成,导入批次间 `setImmediate` 让出事件循环,避免阻塞同进程内其他任务;API 进程与同步进程分离,导入期间在线查询不受影响(WAL 读旧快照)。

**异步化清单**:
- 翻译回填 worker ⬜:与国家同步共存,coordinator 空闲时分批跑,短事务提交,服务不可用时不推进游标。
- 住宅富化 ⬜(进行中):Overture buildings 探测已从 2 网格/1000 点提升到 24 网格/15000 点(residential-buildings-v3),随各国下一轮同步生效;后续可再考虑 OSM building 标签二次标注。
- IP 实时查询 ✅:带 20s 超时,失败立即降级本地池,不拖死请求。
- 前端 ✅:AbortController 取消旧请求;预取与生成并行;位置下拉防抖 280ms。

## 3. 初始化提速(45-70 分钟/国 → 目标 <10 分钟/国)

| 优化 | 状态 | 收益 |
|---|---|---|
| Overture bbox 谓词下推(row-group 剪枝) | ✅ 已在导出器 | 扫描量降一个数量级 |
| 读 bbox 坐标列代替 WKB geometry + spatial 扩展 | ✅ | 少读大 BLOB 列 |
| pyosmium C++ KeyFilter 预过滤 | ✅ | 99%+ 对象不进 Python 回调 |
| >1GB PBF 用 sparse_file_array 位置索引 | ✅ | 内存峰值可控(11GB 机器安全) |
| curl + IPv4 直连下载、断点续传、复用当日原始包 | ✅ | 下载稳定性 |
| **翻译与导入解耦**(导入零翻译请求) | ⬜ | 消除每国 20-40 分钟串行 HTTP,最大单项 |
| importer 网格分组修复(空 locality 不再塌缩) | ⬜ | Geofabrik 国家填充率 5-10 倍(顺带修数据量) |
| 每国导入后 `wal_checkpoint(TRUNCATE)` | ⬜ | WAL 不再累积(实测曾达 285MB) |
| 下一国下载与当前国导入流水线并行 | 🔭 | 两核机器收益有限,暂缓 |

预期:Overture 国家 3-8 分钟/国(纯扫描+导入),Geofabrik 国家受下载带宽支配(RU 4.2GB 除外均 <15 分钟),27 国全量初始化从 1-2 天压缩到 3-5 小时。

## 4. 同步管线稳健性

- 幂等:datasetId = 源版本 + 内容校验和 + 导入修订号;重复触发自动跳过未变化国家。✅
- 断点:每国独立提交;进程被杀后孤儿任务标记 interrupted,锁自动回收,下轮继续。✅
- 门禁:行数/行政区覆盖/降幅阈值,不达标保留旧快照。✅
- 容量:每步前预算检查(当前用量+预估临时+预估 DB 增量),软限 40GiB 停扩容,硬限 45GiB 中止。✅
- 观测:`[address-sync] {国家} discover/materialize/import/ready` 结构化日志;`/api/v1/data-health` 暴露每国水位。✅
- 回退:失败国家次日优先,连续失败退避;手动 API 可即时补跑单国。✅

## 5. 数据质量硬约束(优化不得破坏)

1. 每国 ≤100,000 行、每 locality/网格 ≤64 行,总量 ≤270 万行。
2. 机构黑名单与住宅证据规则在导出、导入、读取三层重复校验。
3. 任何采样/过滤改动必须 bump IMPORT_REVISION 强制重导,禁止新旧混池。
4. 50GB 为绝对上限,预算含 DB+WAL+暂存+备份。

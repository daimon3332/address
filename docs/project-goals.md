# 项目目标

一句话:自托管的 27 国真实地址生成器 —— 地址骨架真实、室内层级合成、三语输出、筛选永远有结果。

## 1. 产品目标

1. **永远出地址**:任何受支持国家、任何筛选组合(州省/城市/邮编/IP 区域),只要该国有数据就必须返回地址;精确匹配不到时按 就近坐标 → 同州省 → 全国 逐级回退并明示回退级别。禁止出现"当前筛选暂无已同步地址"作为最终态。
2. **地址可信**:道路、行政区、坐标、邮编来自真实开放数据(Overture/OSM),每条记录保留来源、许可、版本、观测时间证据;机构地址(政府/学校/医院/银行等)全链路黑名单过滤。
3. **室内层级合成**:门牌号/楼栋/单元/房间可合成并明确标记(`synthetic`),既保证隐私合规(不指向真实住户),又保证表单可用性。中国地址额外合成小区名。
4. **三语一致**:原文 / English / 简体中文三份地址,组件级对应,数字与邮编跨语言不变。
5. **住宅模式**:仅当地址有独立住宅用途证据(OSM building 标签等)才进入住宅池;证据不足的国家不展示住宅模式。
6. **配套测试资料**:姓名/生日/电话/邮箱/职业/薪资/UA 跨字段一致;银行卡仅 Stripe sandbox fixture。

## 2. 技术目标

| 维度 | 目标 |
|---|---|
| 架构 | 单服务器自托管(`/root/address`),零云函数、零云数据库依赖 |
| 存储 | SQLite WAL + RTree,单文件;数据目录 <40GB,硬停 45GB,绝对上限 50GB |
| 性能 | 池内生成 p95 <100ms;前端预取队列点击即出 |
| 初始化 | 27 国全自动连续初始化,断点续跑,重启自愈 |
| 同步 | 每天 1 国、每国 30 天周期,失败优先重试,无人值守 |
| 隐私 | IP 不落库;正常生成 0 外部请求 |

## 3. 边界(明确不做)

- 不做全球地址镜像:每国 ≤10 万条限量池,不追求全量。
- 不承诺门牌级投递真实性:门牌/室内层级可合成,用于表单测试而非实际邮寄。
- 不依赖付费 API:Google/有道翻译走免费通道,失败降级保源文。
- 不修改服务器系统层(systemd/crontab/全局包),一切限定在 /root/address。
